## Controle de Versão

A ferramenta de controle de versão adotada no projeto foi o
[Git](https://git-scm.com/), sendo que o [Github](https://github.com)
foi utilizado para hospedagem do repositório.

O projeto segue a seguinte convenção para o nome de branches:

- `main`: versão estável já testada do software
- `unstable`: versão já testada do software, porém instável
- `testing`: versão em testes do software
- `dev`: versão de desenvolvimento do software

Quanto à gerência de issues, o projeto adota a seguinte convenção para
etiquetas:

- `documentation`: melhorias ou acréscimos à documentação
- `bug`: uma funcionalidade encontra-se com problemas
- `enhancement`: uma funcionalidade precisa ser melhorada
- `feature`: uma nova funcionalidade precisa ser introduzida

## Gerenciamento de Projeto

### Metodologia Ágil 

Para o desenvolvimento deste projeto, a equipe adotou o framework Scrum como metodologia ágil. Conforme destacam Amaral, Fleury e Isoni (2019, p. 68), o Scrum proporciona benefícios relevantes, tais como:

- definição clara dos resultados esperados;
- estabelecimento de ritmo e disciplina na execução;
- atribuição precisa de papéis e responsabilidades (Product Owner, Scrum Master e Development Team);
- fortalecimento da autonomia e do engajamento dos membros;
- compartilhamento colaborativo do conhecimento;
- ambiente favorável à crítica construtiva direcionada às ideias, e não às pessoas.

###  Processo de Trabalho 

O gerenciamento das atividades é realizado por meio do GitHub Projects, estruturado em listas que refletem o fluxo de trabalho ágil:

- Backlog: corresponde ao Product Backlog, reunindo todas as tarefas identificadas ao longo do projeto.
- To Do: representa o Sprint Backlog, ou seja, as atividades planejadas para o ciclo atual.
- In Progress: tarefas em andamento, já iniciadas pela equipe.
- Done: atividades concluídas, validadas em testes e revisões, prontas para entrega ao usuário.

Esse fluxo é visualizado em um quadro Kanban, link: https://github.com/orgs/ICEI-PUC-Minas-PMV-ADS/projects/2651 que permite acompanhar o progresso das tarefas de forma transparente e colaborativa.
